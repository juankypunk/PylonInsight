# Legacy BMU Event CSV Format

## Purpose

This document describes the CSV format generated by BatteryView 3.9.4 when exporting the event log from first-generation Pylontech Battery Management Units (BMU).

---

## Supported Device

The format described in this document has been validated using the following hardware:

```text
Device name         : bmu
Board version       : HP0115SV10R01
Main Soft version   : B52.2.0
Soft version        : V3.2
```

BatteryView version:

```text
3.9.4
```

---

| File               | Encoding        |
| ------------------ | --------------- |

| event_detailed.txt | UTF-16LE        |

## File Structure

The file consists of:

* Header
* CSV column names
* Event records
* Footer

Unlike `history.csv`, this file stores event-driven snapshots rather than periodic samples.

---

## Header

The first line identifies the export type.

Example:

```text
datalist,event,0
```

### Header inconsistency

BatteryView omits the `Date` column from the CSV header.

The exported header is:

```text
Item,Time,Vo(mV),Tempr,Tlow,Thigh,Vlowest,Vhighest,Volt.St,Temp.St,Events,BatEvents
```

However, each data record contains both **Date** and **Time**.

Applications should interpret the effective header as:

```text
Item,Date,Time,Vo(mV),Tempr,Tlow,Thigh,Vlowest,Vhighest,Volt.St,Temp.St,Events,BatEvents
```

---

## Timestamp

The event timestamp is composed of:

* Date
* Time

Applications should reconstruct a single timestamp by combining both fields.

---

## Event Model

Each record represents the operating state of the BMU when an event occurred.

Unlike `history.csv`, records are generated only when BatteryView records an event.

---

## Primary Key

The recommended natural key is:

```text
(Date, Time)
```

The `Item` field is only a sequential index inside the exported file and must not be used as a persistent identifier.

---

## Relationship with History

`history.csv`

* periodic sampling
* fixed interval

`event.csv`

* event-driven sampling
* irregular interval

---

## Known Event Codes

Observed codes include:

* BHV
* BVNOR
* PHV
* PVNOR
* IDLE
* SYSERR

The exact semantics of these codes are currently under investigation.
