# XHB_BMU_NT Event CSV Format

## Purpose

This document describes the CSV format generated by BatteryView 3.9.4 when exporting the event log from XHB_BMU_NT modules.

---

## Supported Device

The format described in this document has been validated using the following hardware:

```text
Device name         : XHB_BMU_NT
Board version       : V30R02C002
Main Soft version   : B52.5.0
Soft version        : V1.2
Module AfeType      : LTC6813
```

BatteryView version:

```text
3.9.4
```

## File Structure

The file consists of:

* Header
* CSV column names
* Event records
* Footer

Unlike `history.csv`, this file stores event-driven snapshots rather than periodic samples.

---

## Header

The first line identifies the export type.

Example:

```text
datalist,event,0
```

### Header inconsistency

BatteryView omits the `Date` column from the CSV header.

The exported header is:

```text
Item,Time,Vo(mV),Tmpr,BTlow,BThigh,BVlow,BVhigh,PT.Tmpr,NT.Tmpr,Ref.Vol,Fan.Pwm,Fan1.Rpm,Fan2.Rpm,Base.St,Volt.St,Tmpr.St,PT.Tmpr.St,NT.Tmpr.St,Err.Code,Events
```

The effective logical header is:

```text
Item,Date,Time,Vo(mV),Tmpr,BTlow,BThigh,BVlow,BVhigh,PT.Tmpr,NT.Tmpr,Ref.Vol,Fan.Pwm,Fan1.Rpm,Fan2.Rpm,Base.St,Volt.St,Tmpr.St,PT.Tmpr.St,NT.Tmpr.St,Err.Code,Events
```

---

## Timestamp

The event timestamp is composed of:

* Date
* Time

Applications should reconstruct a single timestamp from both fields.

---

## Event Model

Each record represents the complete operating state of the module when an event occurred.

The XHB generation exports a richer dataset than the legacy BMU, including:

* additional temperature sensors
* reference voltage
* fan-related telemetry
* independent status fields for each temperature sensor

---

## Additional Telemetry

Compared to the legacy BMU, the following fields are available:

* PT.Tmpr
* NT.Tmpr
* Ref.Vol
* Fan.Pwm
* Fan1.Rpm
* Fan2.Rpm
* PT.Tmpr.St
* NT.Tmpr.St

These fields reflect the newer hardware architecture and provide additional diagnostic information.

---

## Primary Key

The recommended natural key is:

```text
(Date, Time)
```

The `Item` field is only a sequential index within the exported file.
